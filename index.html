1. Boost.Asio 기반 네트워크 서버 프로젝트

프로젝트 개요

  • Boost.Asio 라이브러리를 이용한 채팅서버 만들기
  • 프로젝트 기간 : 2025.01 ~ 진행중
  • Github : https://github.com/RyuYoungJoon/ProjectYJ
  • 프로젝트 상세 문서 : 프로젝트 상세 문서 보기

🛠️ 사용된 주요 기술

  • Boost.Asio - 비동기 네트워크 통신 및 이벤트 루프
  • Google Protobuf - 효율적인 패킷 직렬화 및 프로토콜 정의
  • 멀티스레드 프로그래밍 - 패킷 라우터 및 세션 관리를 위한 스레드 풀 구조
  • 메모리 풀링 - ObjectPool, PacketPool을 통한 동적 할당 최소화
  • WinAPI - 클라이언트 UI 구현
  • Plog 라이브러리 - 멀티스레드 환경 로깅 시스템
  • INIReader - 설정 파일 파싱 및 관리

ServerCore 구조

  • Boost.Asio 기반 비동기 TCP 통신 구조 설계 및 구현
  • ObjectPool, PacketPool 기반 메모리 풀링 시스템 도입
  • 멀티 스레드를 이용한 패킷 라우터 및 세션 관리 구조 설계
  • TaskQueue를 통한 상태 관리 (Send, Recv, Disconnect)

  • 패킷 자동화 툴 개발
  • 메모리 누수 감지 및 방지
  • 로그인, 로비, 채팅 시스템 개발

  • ClientManager를 통한 유저 관리 로직 구성

핵심 개발 내용

🔧 가디언 1차 잠재능력 복구 기능

📋 도전 과제

  • 가디언 강화 시 잠재능력이 랜덤하게 변경되는데, 유저들이 이전 능력치로 되돌리고 싶어 하는 니즈 발생
  • 기존 DB 구조에서는 이전 능력치 이력 관리 불가
  • CS 처리 시 수동 복구 작업의 높은 오류율과 시간 소요

🛠️ 해결 방법

  • 잠재능력 히스토리 DB 설계
    • 신규 테이블 GuardianTraitHistory 생성 (UserID, GuardianID, BeforeStep, AfterStep, Timestamp)
    • 강화 시마다 이전 능력치를 자동 저장
    • Stored Procedure로 원자적 저장/조회/복구 처리
  • 복구 시스템 구현
    • 복구 요청 패킷 처리 및 유효성 검증 (소모 아이템, 쿨타임 체크)
    • Redis 기반 복구 쿨타임 관리 (24시간 1회 제한)
    • 복구 실패 시 롤백 처리 및 에러 로그 자동 전송
    • 잠재 능력 강화 시 이전 능력치 캐싱하여 DB 조회 최소화
  • 로그 시스템 구축
    • 복구 전/후 능력치 데이터 상세 로깅
    • CS 팀용 조회 쿼리 제공 (유저별 복구 이력 추적)
💡 기술적 포인트
  • DB 트랜잭션 관리: 복구 과정을 하나의 트랜잭션으로 묶어 중간 실패 시 자동 롤백, 데이터 일관성 보장
  • 쿼리 최적화: 히스토리 조회 시 인덱스 활용 (UserID + Timestamp) 으로 평균 조회 시간 150ms → 15ms 개선
✅ 결과
  • 부담 없는 강화로 강화에 필요한 재화 소모 증가
  • 복구 실패율 0.1% 미만 유지, 안정적인 서비스 제공

🔧 채팅 말풍선 배경 선택 기능

📋 도전 과제

  • 채팅 커스터마이징 요구에 대응하기 위한 신규 아이템 카테고리 추가 필요
  • 일반 아이템과 기간제 아이템을 구분하여 관리해야 하는 복잡성
  • 기간제 아이템의 만료 처리 및 알림 시스템 구현
  • 경쟁전 보상과 연동하여 자동 지급되는 시스템 구축
  • 기존 채팅 시스템에 영향 없이 기능 추가

🛠️ 해결 방법

  • DB 구조 설계
    • 신규 테이블 AccountChatBG 생성 (UserID,ItemID, AcquireDate, ExpireDate, IsActive)
    • 기간제 판단 로직: ExpireDate가 NULL이면 영구, 값이 있으면 기간제
    • 활성화 중인 말풍선 1개만 선택 가능하도록 IsActive 플래그 관리
  • 아이템 지급 시스템
    • 상점 구매, 이벤트 보상, 경쟁전 정산 등 다양한 경로 통합 지급 함수 구현
    • 중복 지급 체크 (영구 아이템은 1개만, 기간제는 기간 연장 방식)
    • 지급 실패 시 보상 우편함 자동 전송 로직
  • 경쟁전 연동 자동 지급
    • 경쟁전 시즌 종료 시 등급별 보상 테이블 참조
    • 배치 작업으로 대상 유저들에게 일괄 지급
  • 만료 처리 시스템
    • 서버 시작 시 만료된 아이템 일괄 비활성화
    • 유저 로그인 시 만료 체크 및 클라이언트 동기화
    • 만료 7일 전, 1일 전 알림 시스템 구현 (스케줄러 활용)
  • 채팅 패킷 확장
    • 기존 채팅 패킷에 ChatBGID 필드 추가
    • 하위 호환성 유지 (ChatBGID = 0이면 기본 말풍선)
    • 클라이언트에서 ChatBGID로 UI 렌더링 처리
💡 기술적 포인트
  • 기간제 아이템 관리: 만료 시각을 UTC 기준으로 저장하여 다국가 서비스 대응, 시간대 문제 방지
  • 알림 스케줄러: DB 스케줄러를 사용하여 매일 새벽 4시 만료 임박 아이템 체크 작업
✅ 결과
  • 런칭 첫 달 말풍선 획득 유저 2,000명, 일평균 사용률 60% 기록
  • 기존 채팅 시스템 성능 저하 없이 안정적으로 통합
